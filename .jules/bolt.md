## 2026-04-12 - OpenAPI Parser Component Schema Conversion Performance Bottleneck
**Learning:** In the `OpenAPIToMCPConverter` class, the `convertComponentsToJsonSchema` method was being called multiple times per operation to generate the `$defs` field for JSON schemas. Because this method iterated over all components schemas and called `convertOpenApiSchemaToJsonSchema` for each, its performance cost grew O(N*M) where N is the number of schemas and M is the number of operations, leading to multi-second delays for large OpenAPI specs.
**Action:** Implemented caching for the `convertComponentsToJsonSchema` result. The `components` definition doesn't change per operation during conversion, so caching the fully converted schema components using a simple property (`this.cachedComponentsSchema`) prevents massive redundant work, reducing processing time for complex specs from 8000ms+ to 25ms.
