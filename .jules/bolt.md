## 2025-05-18 - Caching OpenAPI Component Schemas Conversion
**Learning:** Found a specific performance bottleneck in how `notion-mcp-server` converts OpenAPI specifications. During the conversion to MCP format, `convertComponentsToJsonSchema` loops over all component schemas for EVERY operation parameter and EVERY response because `resolveRefs` defaults to false and does not cache the non-ref root components. For massive OpenAPI specs (thousands of operations and components), this results in O(N * M) performance impact where it computes identical schemas repeatedly, blocking the event loop on startup.
**Action:** Implemented an explicit caching mechanism (`this.componentsCache`) for `convertComponentsToJsonSchema` so that component schemas are converted exactly once. This small change reduces execution time of large specs by 400x (seconds down to milliseconds).
