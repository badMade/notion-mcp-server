## 2025-04-13 - Cached component schema conversion during openapi to MCP mapping
**Learning:** During OpenAPI translation into MCP tools, the conversion step processes schemas multiple times per endpoint (for input parameters, body schemas, and return types). For APIs with large object definitions (e.g. nested schemas with lots of definitions), mapping the components dict (`openApiSpec.components.schemas`) on each call to `$defs` leads to thousands of redundant iterations and object allocations. This creates a severe performance bottleneck.
**Action:** Implemented schema conversion caching (`_cachedComponentsSchema`) in `OpenAPIToMCPConverter` for component dictionaries, so it's calculated only once per spec. Always cache high-cardinality component/dictionary mapping conversions if they're used to resolve references (`$refs`) internally across multiple endpoints.
