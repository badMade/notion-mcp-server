## 2024-07-23 - Prevented Secret Leakage in Error Logs
**Vulnerability:** The `HttpClient` and `MCPProxy` classes were catching errors from the `openapi-client-axios` client and dumping the entire `error` object into `console.error`. This is a critical information disclosure vulnerability. When `axios` fails, its `error` object contains the `error.config` object, which includes the request headers (including `Authorization: Bearer <token>`). These secrets were then leaked into standard error/logs, posing a high security risk.
**Learning:** This codebase uses Axios for HTTP requests, and its error objects are notoriously verbose, containing the original request config including sensitive headers. Logging the whole error object blindly is a common but dangerous pattern when dealing with authentication tokens.
**Prevention:** Always log specific, safe properties of error objects (like `error.message`, `error.response?.status`) rather than the entire object, especially in network clients where request configurations often contain secrets.
