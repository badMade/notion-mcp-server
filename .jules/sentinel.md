## 2025-02-27 - Path Traversal Vulnerability in File Uploads
**Vulnerability:** The HTTP client for handling file uploads (via `fs.createReadStream`) lacked path validation, exposing the application to arbitrary file reads (path traversal) based on malicious parameter input.
**Learning:** This existed because file paths obtained from the operation parameters were directly passed to `fs.createReadStream` without checking if they were restricted to a safe, intended directory. Additionally, blindly splitting environment variables (like `MCP_ALLOWED_DIRECTORIES`) on commas without checking for empty strings or trailing commas can unintentionally whitelist directories like the working root (`.`).
**Prevention:** Implement path traversal validation using `path.resolve` and `path.relative` to ensure any incoming file path is strictly bounded within explicitly allowed directories (`MCP_ALLOWED_DIRECTORIES`). Always filter out empty string paths when parsing comma-separated directory lists.
