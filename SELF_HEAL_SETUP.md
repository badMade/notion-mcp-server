# Self-Heal Automation Setup

This document describes the auto-repair and auto-scheduling pipeline designed for the codebase. It uses telemetry and autonomous triggers to run linting, snapshot generation, type checking, and format fixing via a standard multi-step pipeline.

## Pipeline Architecture

The system consists of these components:
- **`scripts/healthcheck.mjs`**: The central gating check verifying build integrity, type safety, and test passes.
- **`scripts/self_heal.mjs`**: The universal repair sequence. It executes:
  1. Clean installation of tooling (`npm ci`).
  2. Auto-formatting (`prettier --write`).
  3. Snapshot updates (`vitest -u`).
  4. Type stub / configuration checks (`tsc`).
  5. Dependency refresh (`npm update`).
  6. Asset regeneration (`npm run build`).
- **`scripts/compute_schedule.mjs`**: Examines git commit frequency over the last 30 days and dynamically computes an optimized cron schedule.

## Workflows and Triggers

1. **Self-Heal Workflow** (`.github/workflows/self-heal.yml`):
   - **Triggers**: Scheduled (telemetry-driven), Workflow Run (on `ci` failure), or Workflow Dispatch (manual).
   - **Behavior**: Checks out the code, runs the self-heal pipeline, and generates an automated PR only if changes safely address the issue (diffs generated and secrets absent).

2. **Compute Schedule Workflow** (`.github/workflows/compute-schedule.yml`):
   - **Triggers**: Weekly on Sunday, or Workflow Dispatch (manual).
   - **Behavior**: Evaluates git telemetry, modifies `.github/self-heal-schedule.yml` and updates the `# AUTO-UPDATED` cron comment in `self-heal.yml`. If the schedule is adjusted, it issues a PR (`[Self-Heal Schedule]`).

## Reviewer Checklist

When reviewing a PR generated by this pipeline, verify the following:
- [ ] Changes are limited to format fixes, snapshot/test updates, stubs, lockfiles, or static build artifacts.
- [ ] No application business logic has been unintentionally altered.
- [ ] No sensitive credentials (API keys, tokens, passwords) are exposed in the diff.
- [ ] The pipeline output has successfully run through `healthcheck.mjs`.

## Manual Overrides

If the automatically computed schedule is inadequate:
1. Open `.github/self-heal-schedule.yml` and manually adjust the `schedule` string.
2. Edit `.github/workflows/self-heal.yml` and replace the cron string next to `# AUTO-UPDATED`.
3. *(Optional)* Pause the `compute-schedule.yml` workflow via the GitHub Actions UI to prevent future overrides.
