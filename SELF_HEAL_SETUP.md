# Self-Heal Auto-Repair Setup

This repository is equipped with a self-healing pipeline designed to automatically fix code drift, dependency updates, type stubs, and formatting issues. The automation acts as a proactive agent that submits pull requests for human review.

## Triggers

The self-heal pipeline runs via three primary triggers:
1. **Scheduled:** Proactively runs on an adaptive schedule based on repository velocity (telemetry).
2. **Reactive:** Triggers automatically if the main `ci` workflow fails on the default branch.
3. **Manual Dispatch:** Can be triggered manually by maintainers via the GitHub Actions UI.

## Self-Scheduling Logic

The pipeline computes its own execution cadence using PR telemetry (merge frequency):
- **High Velocity:** Every 6 hours
- **Active Velocity:** Every 12 hours
- **Standard Velocity:** Daily (default)
- **Low Churn:** Weekly

This computation is handled by `.github/workflows/compute-schedule.yml` running `scripts/compute_schedule.mjs`.

## Manual Overrides

To manually override the schedule:
1. Edit `.github/self-heal-schedule.yml`
2. Set your desired cron expression.
3. Commit and push the changes. The oscillation guard will respect recent manual edits.

## Reviewer Checklist

When reviewing a PR generated by the self-heal pipeline, ensure the following:
- [ ] No recursive deletions or unexpected file removals.
- [ ] No logic changes (only formatting, snapshots, dependencies, or type stubs).
- [ ] No secrets or credentials were automatically committed.
- [ ] Tests and build pass successfully (the `healthcheck.mjs` gate validates this prior to PR creation).

*Note: This pipeline is fully idempotent. If a fix fails or no diff is produced, the pipeline exits safely.*
