# Self-Heal Pipeline Setup

This project uses an automated self-healing CI pipeline designed to fix simple code drifts, test snapshots, dependency lockfile issues, and formatting problems without manual intervention. The pipeline creates a PR for human review.

## Architecture

*   **`scripts/healthcheck.mjs`**: Verifies that the codebase is healthy by running linting, tests, and builds.
*   **`scripts/self_heal.mjs`**: A 6-step idempotent repair process that runs when drifts are detected.
*   **`scripts/compute_schedule.mjs`**: Computes an optimal run schedule based on repository telemetry.
*   **`.github/workflows/self-heal.yml`**: The main GitHub Actions workflow that orchestrates the self-heal pipeline and PR creation.
*   **`.github/workflows/compute-schedule.yml`**: The workflow that recalculates the optimal schedule and auto-updates `.github/self-heal-schedule.yml` and `self-heal.yml`.
*   **`.github/self-heal-schedule.yml`**: Stores the current telemetry-derived schedule.

## Triggers

The Self-Heal Pipeline can be triggered in three ways:

1.  **Scheduled**: Automatically runs based on the cadence defined in `.github/self-heal-schedule.yml`.
2.  **Reactive (CI Failure)**: Automatically triggered whenever the main `ci` workflow fails.
3.  **Manual**: Can be manually dispatched via the GitHub Actions UI.

## Repair Steps

The pipeline performs the following idempotent repair steps:

1.  **Rebuild/reinstall**: Cleans dependencies and installs (`npm ci`).
2.  **Lint/format**: Auto-fixes lint and formatting issues (`npx eslint --fix . && npx prettier -w .`).
3.  **Snapshot updates**: Updates test snapshots (`npx vitest run -u`).
4.  **Type stubs**: Refreshes type stubs and builds (`npm run build`).
5.  **Dependencies**: Performs a minor update of dependencies (`npm update`).
6.  **Assets**: Regenerates static assets and final builds.

After each step, the `healthcheck.mjs` script runs. If the codebase becomes healthy and there is a git diff, the script immediately exits and a PR is created.

## Manual Overrides

To manually override the computed schedule:

1.  Edit `.github/self-heal-schedule.yml`.
2.  Change `SELFHEAL_SCHEDULE` to your desired cron string.
3.  Ensure you leave the `# AUTO-UPDATED` marker on that line, as the sed script relies on it.
4.  Update the `RATIONALE` string to note the manual override.
5.  Commit and push to `main`.

## Reviewer Checklist

When reviewing a PR generated by the `self-heal` pipeline, please check:
- Is the PR title prefixed with `[Self-Heal ...]`?
- Are the artifact logs (`pre-check.log`, `repair.log`, `post-check.log`) attached and clean?
- Are the code changes strictly related to formatting, snapshot updates, or minor dependency bumps?
- Ensure no secret leakage or application logic modification occurred.
